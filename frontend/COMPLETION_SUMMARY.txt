  QUICK START
════════════════════════════════════════════════════════════════════════════════

1. Install dependencies:
   npm install

2. Set environment variable (.env.local):
   NEXT_PUBLIC_API_BASE=http://localhost:8000

3. Start development server:
   npm run dev

4. Open http://localhost:3000

5. Verify API is returning data by checking Network tab

  KEY COMPONENTS EXPLAINED
════════════════════════════════════════════════════════════════════════════════

DynamicDashboard.tsx
  - Main container component
  - Orchestrates all API hooks
  - Renders 5 dashboard sections
  - Calculates real-time statistics
  - Handles loading and error states

StatCard.tsx
  - Displays single metric
  - Reusable for any stat
  - Shows label, value, explanation
  - Three visual variants

SectionCard.tsx
  - Content display component
  - Used for services, devices, etc.
  - Supports images, badges, details
  - Responsive grid layout

useServices Hook
  - Fetches from /api/v1/services
  - Caches for 30 seconds
  - Returns { services, isLoading, isError }

useDevices Hook
  - Fetches from /api/v1/devices
  - Caches for 30 seconds
  - Returns { devices, isLoading, isError }

useTeam Hook
  - Fetches from /api/v1/team
  - Caches for 60 seconds
  - Returns { team, isLoading, isError }

useProjects Hook
  - Fetches from /api/v1/projects
  - Caches for 30 seconds
  - Returns { projects, isLoading, isError }

ADD A NEW FIELD

  2. Update component to display field

CREATE NEW SECTION

  2. Import in DynamicDashboard.tsx
  3. Add section JSX
  4. Done!

════════════════════════════════════════════════════════════════════════════════
  TROUBLESHOOTING
════════════════════════════════════════════════════════════════════════════════

Dashboard shows "No items found"
  → Check backend is running
  → Check API endpoint returns data
  → Check Network tab in DevTools

API calls fail / CORS error
  → Check API base URL in .env.local
  → Check backend has CORS enabled
  → Check backend is accessible

Data not updating
  → SWR revalidates every 30-60 seconds
  → Check if data actually changed in backend
  → Try refreshing page

Images not loading
  → Check image URLs in API response
  → Check images are publicly accessible
  → Check image format is supported

For more help, see GETTING_STARTED.md → Debugging section
